# AGENTS.md

## Scope

This file applies to everything under `docs/`.

## Documentation purpose

Docs should support:
1. technical understanding
2. recruiter-facing positioning
3. business-facing readability for mining analytics roles

## Tone and framing

Write clearly and concretely.
Prefer:
- planning
- management control
- KPI monitoring
- performance analytics
- BI
- decision support

Do not overframe the repo as:
- a pure quant experiment
- a mine-engineering solution
- a live enterprise platform

## Required documentation behavior

When documenting a new dataset, mart, or analytical layer, state:
- purpose
- grain
- columns or major fields
- units
- intended dashboard use
- limitations

## README consistency

Keep README and docs aligned.
If architecture or outputs change materially, update docs accordingly.
