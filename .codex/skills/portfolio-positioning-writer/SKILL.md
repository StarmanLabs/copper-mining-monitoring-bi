# Skill: portfolio-positioning-writer

## Scope

This is a repository-local skill for `copper_minning_risk_model`.

Use it only for this project and for recruiter-facing, manager-facing, and portfolio-facing writing that explains the repository clearly and credibly.

## Current repository context

This repository is currently positioned as:

- a mining planning and performance analytics portfolio project
- a business-facing mining BI and decision-support workflow
- a stronger fit for economics + data + mining domain adaptation than for mine engineering or geology depth

The skill should preserve that positioning and keep public documentation aligned with the actual state of the repository.

## Purpose

Write recruiter-facing and manager-facing documentation for a mining analytics portfolio aimed at planning, management control, BI, and economics-facing roles.

## When to use

Use this skill when the task involves:
- README rewrites
- docs refresh
- portfolio summaries
- recruiter-facing explanations
- role alignment messaging

## Audience

Primary audience:
- mining hiring manager
- planning lead
- BI lead
- analytics reviewer
- business-facing technical recruiter

## Positioning rules

Always reinforce:
- mining planning
- performance analytics
- management control
- BI
- KPI monitoring
- decision support
- economics + data profile

Do not overemphasize:
- pure quant finance
- Excel migration
- Monte Carlo as the whole story
- mine-engineering claims

## Writing rules

- be concrete
- be honest
- explain what the repo does first
- explain limitations clearly
- avoid hype
- connect outputs to business decisions

## Completion checklist

Before finishing:
- confirm docs help a recruiter understand the repo quickly
- confirm language matches actual repo capability
- confirm the economics profile is visible
- confirm the writing is specific to this repository rather than a generic mining analytics portfolio template
